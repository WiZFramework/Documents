０１．はじめに

０１－０１．【はじめに】と開発ツール

　このドキュメント目的は新しいプログラミング言語を作成することです。
　具体的に述べましょう。
　たとえばPHPを作成することを考えてみましょう。
　PHPはphpブロックにはさまれた部分は実行を行いそれ以外は標準出力をするプログラム言語です。
　以下の様なphpファイルがあった場合 
<!DOCTYPE html>
<html lang="ja">
<head>
<meta charset="utf-8">
<title>CGI言語作成（2018年）</title>
</head>
<body>
こんにちはHTML<br />
<?php
echo "こんにちはPHP";
?>
</body>
</html>
　ブラウザの表示は以下のようになります。 
こんにちはHTML
こんにちはPHP
　すなわち、phpブロック内ではecho "こんにちはPHP";という構文が実行され、出力されます。
　ここでechoという構文はPHPでは出力するという意味になりますが、言語によってはprt "こんにちは";が出力になるかもしれません。
　これはプログラム言語の文法によります。

新しい言語とは

　PHPにおけるechoをprtに置き換えた場合、これは、新しい言語になります。
　例えば、PHPブロックをoreというブロックに変え、以下のように記述すれば 
<!DOCTYPE html>
<html lang="ja">
<head>
<meta charset="utf-8">
<title>CGI言語作成（2018年）</title>
</head>
<body>
こんにちはHTML<br />
<?ore
prt "こんにちはORE";
?>
</body>
</html>
　とすれば、一番単純なoreプログラミングの完成です。
　もちろん、PHPのように、ブロック以外は出力にしなくてもかまいません。文書全てを実行構文として 
%str<hdoc:HDOC
<!DOCTYPE html>
<html lang="ja">
<head>
<meta charset="utf-8">
<title>CGI言語作成（2018年）</title>
</head>
<body>
こんにちはHTML<br />
HDOC;
prt %str;
prt "こんにちはORE";
%str<hdoc:HDOC
</body>
</html>
HDOC;
prt %str;
　として 
%str<hdoc:HDOC
から 
HDOC;
を、ヒアドキュメントの記述のように扱えれば、PHPの仕様のようにはしなくてもいいと思います。
　この場合%strは変数です。PHPは$で変数は始まりますが、%で始まるのようにした場合です。
　あるいは、PythonのようにC言語系とは、まったく違った言語体系にしてもいいと思います。

このドキュメントの目的

　このドキュメントの目的はお勉強です。僕自身の勉強でもあり、C/C++を極めたいと思っている人のお勉強のためでもあります。ですから、このドキュメントに沿っていろいろ作成したからといって、いま、あなたがプロのプログラマーだったとして、開発現場で取り組んでいる何かの役に立つわけではありません。むしろ、現場の弊害になるような記述方法が随所に入っているかもしれません。
　でもひょっとしたら、直接は役にはたたなっけどこういう書き方があるのかという一つの指針になるかもしれません。
　僕自身、C/C++に長くかかわってきて、決して模範となる書き方ではないけれど、ある程度、経験則的な記述の癖があります。
　そういった癖があちこちに出現します。それを読んでいただく、あるいは記述して実行していただくことで、何らかの指針が伝えられればいいなと思っています。

そしてどこに向かうのか

　最終目的は現在はっきりているわけではありません。まずはインタプリタ言語を開発します。そこまでは目標とします。
　次のステップは擬似アセンブラ化です。Javaとかと同じですね。そこまで行ければまあ満足でしょう。
　さらに言えば、それをサーバーでCGIとして動くところまで行ければ、ひょっとしたら誰かが使ってくれる言語になるかもしれません。

作成ツール

　さて、新しい言語を作る環境として、なにが一番取り込みやすいでしょうか？
　もちろんＷＥＢサーバーを立て（ローカルサーバでもよいけども）、Linux環境で作成するのもいいのですが、ちょっとハードルが高いような気がします。
　そこでまずはVisuaStudioの環境で作成するWindowsアプリとして作成しましょう。
　将来的にはLinuxで動くあるいはCGI化を考えるとして、ビルドしたり実装したりを繰り返す作業になりますので、できるだけ開発しやすい環境が良いと思います。
　その点,VisuaStudioであればエディタやコンパイラもGUIで使いやすいです。
　最低限必要な作成環境、使用ツールは以下です。 
１、VisualStudio2015もしくは2017のC/C++プログラム環境（コンソールプログラミング）
２、Windows版、Flex
３、Windows版、Bison
　上記ツールを確認しましょう。

公開する内容について

　今のところVisualSstudioのソリューションなどは公開、あるいはダウンロードできるようにするつもりはありません。
　冒頭述べましたように、このプロジェクトはお勉強のためのものです。
　その代り、このドキュメントに則してコードを記述（コピペでもよい）すれば、その項目で実装した内容が実装できるはずです。
　もし動かないとすれば、記述が間違っているか、あるいは設定なども結構変わりますので、そのあたりを読み飛ばしているか、です。
　そういう場合は、少し前に戻ってプロジェクトを書き直してみましょう。
　それでは、次項では簡単な計算機を作成します。
